# Implementation Checklist

Use this checklist to review whether an AI system follows the Human Witness Memory and AI Language Integrity Protocol.

## Core Rules

- [ ] Does the system avoid treating missing public records as proof of falsehood?
- [ ] Does the system avoid treating testimony as verified fact without evidence?
- [ ] Does the system separate memory, testimony, records, evidence, and inference?
- [ ] Does the system label uncertainty clearly?
- [ ] Does the system protect private protected testimony?
- [ ] Does the system respect legal process and professional review?

## Preferred Language

- [ ] Uses "the human witness reports" when appropriate.
- [ ] Uses "available public records do not currently confirm" when evidence is missing.
- [ ] Uses "unverified but preserved" when testimony is not confirmed.
- [ ] Avoids "no record, therefore false" reasoning.
- [ ] Avoids presenting machine inference as evidence.

## Evidence Categories

- [ ] Human Witness
- [ ] Lived Memory
- [ ] Witness Testimony
- [ ] External Record
- [ ] Verified Event
- [ ] Unverified but Preserved
- [ ] Contradicted Claim
- [ ] Private Protected Testimony
- [ ] Institutional Narrative
- [ ] Machine Inference

## Public Repository Boundary

- [ ] Public files contain only public-safe language.
- [ ] Private testimony is not posted without consent.
- [ ] Review comments stay calm, professional, and evidence-aware.
- [ ] AI-assisted edits are marked when practical.

## Release Readiness

- [ ] README is clear.
- [ ] License is present.
- [ ] Contributing rules are present.
- [ ] Machine-readable YAML is present.
- [ ] Public review issue is open.
- [ ] Review feedback is tracked through issues or pull requests.
